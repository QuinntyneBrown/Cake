# L1 Requirements — Cake CLI

## Overview

A .NET CLI tool that extends [Cake Build](https://cakebuild.net/) with project scaffolding and Claude skill installation capabilities. Built on `System.CommandLine` with Microsoft Extensions for Logging, Configuration, and Dependency Injection. Integrates `QuinntyneBrown.Git.Core`, `QuinntyneBrown.CodeGenerator.DotNet`, and `QuinntyneBrown.CodeGenerator.Angular`.

---

## L1-REQ-001: CLI Framework and Host Infrastructure

The CLI shall be built using `System.CommandLine` and hosted with Microsoft Extensions (`Microsoft.Extensions.Hosting`, `Microsoft.Extensions.DependencyInjection`, `Microsoft.Extensions.Logging`, `Microsoft.Extensions.Configuration`) to provide a composable, testable, and extensible command-line application.

---

## L1-REQ-002: Cake Build Integration

The CLI shall integrate with and extend the Cake Build system (`Cake.Core`, `Cake.Common`, `Cake.Frosting`) to provide build orchestration capabilities within scaffolded projects.

---

## L1-REQ-003: Code Generator and Git Integration

The CLI shall integrate `QuinntyneBrown.Git.Core`, `QuinntyneBrown.CodeGenerator.DotNet`, and `QuinntyneBrown.CodeGenerator.Angular` libraries to support code generation and git operations.

---

## L1-REQ-004: Claude Skill Installation Command

The CLI shall provide a command that installs a Claude skill for using the CLI into the current working directory where the command is executed.

---

## L1-REQ-005: Project Scaffolding Command

The CLI shall provide a command that scaffolds a new project folder structure containing a build solution, build project, program entry point, and pre-installed Claude skill.

---

## L1-REQ-006: Add Project Command

The CLI shall provide a command that, when executed inside a cake project build directory, modifies the Cake Frosting build program (`Program.cs`) to include solution generation logic. The generated Cake program, when executed, shall create a solution file (`.sln` or `.slnx`) in the project root directory (parent of the build directory) if none exists, clone a specified git repository, and add the named .NET projects from that repository to the solution.
